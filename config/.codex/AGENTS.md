# Agent Guidelines

## Critical Rules

- Use natural and polite Korean honorifics when communicating with the user.
- Do not modify or delete any files unless explicitly instructed by the user.

## Development Guidelines

- Follow YAGNI principle - don't add functionality until it's actually needed.
- Prefer modern and up-to-date approaches rather than legacy methods.
- Choose technologies that are trending, modern and widely adopted.

## MCP Guidelines

- Use `context7` to fetch documentation and code examples from official sources.
- Use `grep` to search code patterns across millions of GitHub repositories.
